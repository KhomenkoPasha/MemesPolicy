2. Какие данные мы собираем

Приложение может обрабатывать следующие категории данных:

2.1. Данные аккаунта

  • данные для входа через сторонние сервисы (например, Facebook/Firebase): идентификатор пользователя, имя/никнейм, фото профиля (если доступно), email (если предоставлен).

2.2. Пользовательский контент (UGC)

  • изображения/мемы, которые вы загружаете или создаёте;
  • подписи/описания/комментарии (если доступны);
  • данные профиля (например, ник, аватар).

2.3. Данные устройства и технические данные

  • версия приложения, версия ОС, язык, модель устройства;
  • данные об ошибках и диагностике (для улучшения стабильности);
  • IP-адрес и общие сетевые данные (для работы интернет-функций).

2.4. Рекламные данные

Если в приложении показывается реклама (например, Google Ad Manager/AdMob), рекламные SDK могут собирать:

  • рекламный идентификатор (Advertising ID) или его аналоги, где применимо;
  • данные о взаимодействии с рекламой (показы/клики);
  • примерное местоположение на основе IP (в зависимости от конфигурации рекламной сети).

3. Разрешения Android и зачем они нужны

  • Камера — чтобы создавать/загружать изображения;
  • Доступ к фото/медиа — чтобы выбрать изображение для публикации или сохранения;
  • Интернет / состояние сети — чтобы загружать контент и работать с сервисами авторизации/рекламы.

4. Как мы используем данные

  • предоставление функций приложения (вход, профиль, публикации);
  • отображение и хранение пользовательского контента;
  • обеспечение безопасности и предотвращение злоупотреблений;
  • улучшение стабильности (диагностика ошибок);
  • показ рекламы и измерение эффективности (если включено).

5. Передача данных третьим лицам

Мы можем передавать данные сервисам, обеспечивающим работу приложения, например:

  • Firebase (Google) — аутентификация, база данных/хранилище, аналитика/краши (если подключено);
  • Facebook — авторизация (если используется вход);
  • Google Ad Manager / рекламные партнёры — показ рекламы.

Эти сервисы обрабатывают данные в соответствии со своими политиками конфиденциальности.

Если в приложении есть удаление аккаунта/контента — используйте его. Если нет — запросите удаление по адресу hompavlik@gmail.com, указав ник/идентификатор аккаунта и, при необходимости, email входа.
